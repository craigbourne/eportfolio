Unit 2: Why Projects Fail and Gathering Requirement

Module overview

This unit focused on translating theoretical understanding into practical Python programming. The content covered programme domain analysis using UML diagrams, particularly use case and state machine diagrams. It examined how actors interact with software and how to model system behaviours and interactions within a system.

Required Reading

  • Rumbaugh, J., Jacobson, I. & Booch, G. (2004) The Unified Modeling Language Reference Manual, Chapters 5 and 6
  • Von der MaBen & Lichter, H. (2002) Modeling Variability by UML Use Case Diagrams
  • Chanda, J., Kanjilal, A., Sengupta, S. & Bhattacharya, S. (2009) Traceability of requirements and consistency verification of UML use case, activity and Class diagram: A Formal approach
  • visual-paradigm.com (n.d) What is a Use Case Diagram?
